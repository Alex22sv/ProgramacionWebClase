Introducción

En esta práctica de laboratorio se estudiarán las diversas formas de aplicar estilos a un documento de marcado (como HTML); aunque la presente se centrará en el uso de las hojas de estilo css como principal administrador de dicho cometido.

¿Qué es css?

Del inglés "Cascading Style Sheets" (Hojas de estilo en cascada), es el estándar utilizado para describir la aparariencia de documentos que derivan de XML, como lo son HTML, XHTML o SVG. CSS determina la manera en como es modelada la información en pantalla. Este está compuesto por una serie de selectores y propiedades, que al juntarlos en base al estándar, se establecen las reglas; como su nombre lo indica, CSS aplica estas reglas de manera estructurada, una despues de otra, por lo que la última de estas definida (si se encuentra al mismo nivel) dentro del documento tiene mayor prioridad que las anteriores.

Propiedades

Una propiedad es aquella que define un aspecto especifico del estilo de un elemento; este puede ser un color de fondo, una fuente especifica, un margen interior o exterior, entre otros muchos. En CSS, estos son definidos dentro de un selector, lo que se conoce como regla CSS. La sintaxis básica de estas es la siguiente:

selector{
 prop1: value1;
 prop2: value2;
 prop3: value3;
}

Propiedades existen muchas, y nombrarlas en está guía de laboratorio sería poco prático, de tal forma que se recomienda la lectura de los siguientes enlaces:

  • Sintaxis CSS (MDN)
  • Referencia CSS (MDN)
  • Referencia CSS (W3Schools)

Selectores

Estos consisten en el conjunto de reglas que determinan la identificación y administración de uno o varios elementos dentro de un documento. Existen diversas formas de escribir un selector, incluso se puede determinar el comportamiento gráfico de un elemento en ciertos estados específicos (cuando hacemos click, o estamos sobre él con el ratón, por ejemplo); pero los mas comunes son los selectores por elemento, clase o id (ojo, no son los únicos).

  • Por elemento: Este basa su criterio en elementos básicos de la estructura html como body o header (etiquetas como tal). Este tipo de estilo dentro de la jerarquía de selección no representa mucho, pero es de gran utilidad para generalizar estilos y realizar plantillas del sitio. Por ejemplo si se quiere generalizar un margen al body de todo el sitio (como es el caso de la presente guía) la estructura de la regla haciendo uso de este selector quedaría de la siguiente forma:

    body{
     margin-left:10%;
     margin-right:10%;
    }

  • Por clase: Este basa su criterio en elementos que han sido marcado con un identificador a traves del atributo class. A diferencia del anterior, este define su seleccion antecediendo un punto al nombre de la clase que modificará: .class{/*Definition*/}. Este tipo de selector dentro de la jerarquía está por encima de los que se basan en etiquetas. Para ejemplificar este selector tomaremos como referencia la siguiente etiqueta:

    <code class="myCode"></code>

    Donde las etiquetas <code></code> ya están definidas por la siguiente regla CSS:

    code{
     color: #2c3e50;
     font-size:1.2em;
    }

    Y se quiere establecer un estilo particular (texto rojo, centrado y de tamaño 2em) a la etiqueta con la clase "myCode". La solución al problema es establecer una regla por medio de un selector por clase, de la siguiente forma:

    .myCode{
     display: block;
     text-align: center;
     color: #e74c3c;
     font-size:2em;
    }

    Generando un resultado en el navegador como el siguiente:

    Soy un código rojo

    Donde es importante recalcar que el estilo del selector por clase sobreescribió al de etiqueta.

  • Por id: Estos a diferencia de los dos selectores anteriores, delimitan sus reglas de estilos a los elementos (teóricamente únicos) que han sido definidos con el atributo id. Su sintaxis se basa en anteponer un numeral al nombre del id a modificar: #id{/*Definition*/}. Este, dentro de la jerarquia es el más importante, por lo que se antepone ante cualquier otra regla o definicion de estilo.

    Se basará en el ejemplo del iniciso anterior, para ejemplificar este selector; donde la nueva etiqueta (<code class="myCode" id="CodeUnq"></code>) estará definida por ambas reglas previas, tanto de etiqueta como de clase.

    En este caso se quiere conservar los valores delimitados por el selector de clase, pero se quiere cambiar el color a verde unicamente a esa etiqueta. Como es de suponer, la solución viene dada por una regla delimitada por un selector de id, definida de la siguiente forma:

    #CodeUnq{
     color: #27ae60;
    }
