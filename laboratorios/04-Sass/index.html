Sass

Preprocesador de CSS

Introducción a Sass.

Sass, del acrónimo en inglés "Syntactically Awesome Stylesheets", es un metalenguaje de Hojas de Estilo en Cascada, el cual extiende e implementan nuevas funcionalidades a CSS. Normalmente un navegador no es capaz de interpretar los estilos declarados en un archivo con estádares Sass, pero haciendo uso de pre-procesadores de código se pueden traducir a archivos con estándares CSS.

Este metalenguaje soporta dos tipos de extenciones de archivos .sass y .scss; cada uno de ellos representa una sintaxis diferente para la escritura de las hojas de estilos en Sass.

.sass
Sintaxis basada en la indentación del código para declarar los bloques de código y separar las reglas y propiedades. 
                        
    #A
        color: red

        #B
            border: 1px solid black

            #C
                font-size: 2em
.scss
Sintaxis más tradicional, que hace uso de llaves ("{ }") para separar bloques, y punto y coma (";") para separar reglas y propiedades. 
                        
    #A{
        color: red;

        #B{
            border: 1px solid black;

            #C{
                font-size: 2em;
            }
        }
    }

Como la sintaxis de .scss es la más parecida a la sintaxis de CSS, es la que se hará uso dentro del desarrollo de la presente guía.

¿Por qué hacer uso de Sass/Scss?

  • Reglas Anidadas:

    Propiedad que permite la anidación de propiedades sin la necesidad de múltiples bloques de reglas separados. Esto permite un código CSS más legible y limpio.

  • Superconjunto de CSS

    Cualquier sintaxis o derivación de Sass es un super conjunto de CSS, por lo que toda regla escrita en CSS que funcione, seguirá funcionando en Sass.

  • Variables:

    A diferencia de las variables estándares de CSS, las variables de Sass permiten mayor flexibilidad al momento de operarlas, cambiarlas, recorrerlas, etc. Por lo que su uso es altamente recomendado para sitios con una mayor complejidad.

  • Mejores operadores:

    A diferencia de los cálculos en CSS, los cuales requerían el uso de la función calc(), los cálculos en Sass son mas intuitivos y sencillos debido que su implementación se simplifica a solo utilizar los operadores aritméticos.

  • Funciones:

    Sass permite la declaración de bloques de reglas CSS como funciones reutilizables, permitiendo la generalización de funcionalidades CSS comunes en sitios complejos.

  • Mixins:

    Son un conjunto de propiedades CSS declaradas una vez, las cuales se reutilizan mezclandolas con cualquier otra definición CSS existente. Esto permite la existencia de temas en distintos "layouts".

Declaración y uso de variables.

La variables utilizadas en Sass/Scss son diferentes de las utilizadas en CSS. Estas inician con el signo de dolar antes del identificador que representan la variable. Además, esta pueden poseer distintos valores y tipos de propiedades.

Para su uso, solo deben invocarse en cualquier definición CSS dentro del documento.

Por ejemplo.

Scss 

                
    /*

        Definición de variables
    
    */

    $primary_color : #3F51B5;
    $secondary_color : #9C27B0;
    $accent_color : #FF5722;

    $standar_margin : 8px;
    $standar_padding : 4px;
    $standar_border : 1px solid $primary_color;

    /*
    
        Uso de las variables
    
    */

    div{
        width: 80%;
        height: 100px;
    
        color: $accent_color;
    
        margin: $standar_margin;
        padding: $standar_padding;
    
        border: $standar_border;
    }

CSS procesado. 

            
    div {
            width: 80%;
            height: 100px;
            color: #FF5722;
            margin: 8px;
            padding: 4px;
            border: 1px solid #3F51B5;
        }

Reglas anidadas.

Las reglas anidadas permiten expresar reglas CSS de forma menos repetitiva y más legible. Naturalmente la mayoria de selectores poseen un carácter de anidación, por lo que este tipo de sintaxis resulta intuitiva al momento de aplicar bloques de reglas CSS. Son utilies al momento de manejar estilos y plantillas complejas.

Scss 

                
    div{
        width: 80%;
        height: 100px;
    
        color: $accent_color;
    
        margin: $standar_margin;
        padding: $standar_padding;
    
        border: $standar_border;

        /*
            Anidación nivel 1
        */
    
        ul{
            list-style: none;
    
            display: flex;
            flex-flow: row wrap;
            
            /*
                Anidación nivel 2
            */

            li{
                background: $secondary_color;
    
                text-decoration: none;
                
                padding: $standar_padding;
                margin: $standar_margin;

                /*
                    Anidación nivel 3
                */
            }
        }
    }

CSS procesado. 

                
    div {
        width: 80%;
        height: 100px;
        color: #FF5722;
        margin: 8px;
        padding: 4px;
        border: 1px solid #3F51B5;
    }
    
    div ul {
        list-style: none;
        display: -webkit-box;
        display: -ms-flexbox;
        display: flex;
        -webkit-box-orient: horizontal;
        -webkit-box-direction: normal;
            -ms-flex-flow: row wrap;
                flex-flow: row wrap;
    }
    
    div ul li {
        background: #9C27B0;
        text-decoration: none;
        padding: 4px;
        margin: 8px;
    }

Operador &

El caracter u operador & es traducido y entendido como el padre de la anidación en la que se encuentra su declaración; este trabajo es realizado por el pre-procesador, y es aplicado como un nuevo bloque de reglas. Dicha funcionalidad resulta útli cuando se aplican pseudo-clases o pseudo-elementos al padre. Por ejemplo:

Scss 

                
    li{
        background: $primary_color;

        text-decoration: none;
        
        padding: $standar_padding;
        margin: $standar_margin;

        /*
            Hace referencia al mismo li
        */

        &:hover{
            background: $secondary_color;
            text-decoration: underline;
        }
    }

CSS procesado 

                
    div ul li {
        background: #3F51B5;
        text-decoration: none;
        padding: 4px;
        margin: 8px;
    }
        
    div ul li:hover {
        background: #9C27B0;
        text-decoration: underline;
    }

Mixins

Los mixins son un conjunto de reglas CSS agrupadas en un bloque, el cual posee como propiedad principal que puede invocarse en cualquier otro bloque de reglas CSS, volviendose altamente reusable y versatil. Además permite el uso de argumentos en su cuerpo, ampliando el uso de los mismos en el diseño de sitios complejos, permitiendo diversos valores en las propiedades o cálculos específicos.

Para invocar cualquier mixin declarado dentro de la hoja de estilos, se hace uso de la directiva @include, seguida de identificador y paramentros del mixin que se desee utilizar. Por ejemplo: 

                
    /*
        Declaracion de los mixin              
    */

    @mixin centerBox(){
        display: flex;

        justify-content: center;
        align-items: center;
    }

    @mixin simetricSpace($distance){
        margin: $distance;
        padding: $distance;
    }

    /*
        Invocación de los mixin
    */

    .box{
        @include centerBox();

        width: 100%;
        height: 100vh;

        @include simetricSpace(20px);
    }

    .itemm{
        object-fit: cover;

        width: 100px;
        height: 100px; 

        @include simetricSpace(5px);
    }

CSS procesado 

                
    .box {
        display: -webkit-box;
        display: -ms-flexbox;
        display: flex;
        -webkit-box-pack: center;
            -ms-flex-pack: center;
                justify-content: center;
        -webkit-box-align: center;
            -ms-flex-align: center;
                align-items: center;
        width: 100%;
        height: 100vh;
        margin: 20px;
        padding: 20px;
    }
        
    .itemm {
        -o-object-fit: cover;
            object-fit: cover;
        width: 100px;
        height: 100px;
        margin: 5px;
        padding: 5px;
    }
