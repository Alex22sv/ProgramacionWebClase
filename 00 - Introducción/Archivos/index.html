Manejo de archivos.

GNU/Linux.

Introducción

La presente guía pretende introducir los conceptos genererales acerca del uso de archivos, especificamente en una distribución del sistema operativo GNU/Linux.

Un archivo es, en escencia, un conjunto de información, agrupada en determinada estructura. Sin importar el dispositivo que se utilize, toda la información en una computadora es almacenada como archivos. De acuerdo a su estructura, los archivos se clasifican en diversos tipos: audio, video, imagen, texto y muchos otros.

Características de los archivos

Todo archivo posee las siguientes características:

Nombre:
Su principal propósito es identificarlo de otros archivos y ayudar al usuario a conocer su contenido.
Extensión:
Es parte del nombre del archivo. Al final del nombre se coloca un punto seguido de unas pocas letras o números que definen la extensión. El uso de una extensiones es opcional en sistemas operativos basados en Unix.
Tamaño:
Es el espacio que el archivo necesita para ser almacenado en el dispositivo.
Tipo:
Indica el programa utilizado para acceder al contenido del archivo.
Fecha de modificación:
Es la última fecha en que el archivo fue modificado o creado.

Extensiones de archivos.

Las extensiones son únicamente un grupo de caracteres que, precedidos de un punto, son agregados al final del nombre del archivo.

En sistemas operativos Linux se agrega el tipo de archivo MIME al inicio de este, de manera que la extensión no es necesaria, sin embargo, la extensión es utilizada como ayuda al usuario para identificar el tipo de archivo. Si este no está claro la extensión es utilizada para determinarlo.Esto púede ocurrir cuando se transfieren archivos desde un sistema operativo Windows a Linux.

Los sistemas operativos Windows se basan únicamente en la extensión para saber el tipo de archivo; para evitar que usuarios modifiquen esta extensión, Windows oculta las extensiones de archivos de manera predeterminada. Esta opción puede ser cambiada en la configuración de usuario. Si se utiliza un sistema Windows se debe tener especial cuidado al crear o renombrar un archivo. Es común que no se pueda acceder a alguno debido a que este posee una extensión incorrecta o no posea ninguna.

Rutas

Cuando se trabaja con comandos, es habitual pasar como parámetros archivos o directorios. Para indicar alguno de estos se utiliza la ruta o path del mismo, la cual que puede ser absoluta o relativa.

A diferencia de Windows, Linux no tiene unidades, por lo que no es necesario utilizar identificadores como A:\, C:\, D:\ ,etc. El sistema de archivos en Linux funciona con nodos, y cada nodo representa un archivo. El nodo principal puede ser accedido a través del caracter "/"; y el resto de particiones, discos duros, unidades de cd, unidades de dvd, o cualquier otro sistema de archivos, se deben acceder desde directorio principal como si fuese un directorio normal.

Las rutas absolutas se caracterizan por empezar siempre desde la raíz (es decir la "/"), y contener todos los directorios que hay desde este punto hasta el archivo o directorio que queremos indicar; sólo es posible escribir de una forma una ruta absoluta.

Todas las siguientes rutas son absolutas:

  • /home/DouglasHdezT/Desktop
  • /home/DouglasHdezT/Documents/Web/Archivos/index.html
  • /home/DouglasHdezT/Images/imagen_generica.jpg

En cambio las rutas relativas nunca empiezan por el caracter / y puede haber varias rutas relativas para el mismo archivo o directorio al que se quiere acceder. Esto es así porque la ruta relativa depende del directorio en el que se está trabajando.

Por ejemplo,si el directorio Imagenes contiene el archivo imagen_generica.jpg, se puede acceder a este archivo con diferentes rutas relativas de acuerdo con las siguientes situaciones:

Desde el directorio "Imagenes"
imagen_generica.jpg
Desde el directorio "DouglasHdezT"
Imagenes/imagen_generica.jpg
Desde el directorio "/"
home/DouglasHdezT/imagen_generica.jpg

Tipos de archivos.

En Linux existen, básicamente, 5 tipos de archivos:

  • Archivos ordinarios: Contienen la información con la que trabaja cada usuario.
  • Enlaces físicos o duros (hard links): Para lograr entender que es un enlace fisico, primero se debe comprender qué es un inodo. Un inodo es una estructura de datos, propia de los sistemas de archivos tradicionalmente empleados en los sistemas operativos tipo UNIX, como es el caso de Linux. Un inodo contiene las características de un archivo regular, directorio, o cualquier otro objeto que pueda contener el sistema de ficheros.

    • Una vez comprendido esto, un enlace duro es un archivo que apunta al mismo contenido almacenado en disco al que apunta el archivo original. Por lo tanto los archivos originales y los enlaces duros dispondrán del mismo inodo, y consecuentemente ambos estarán apuntando hacia el mismo contenido almacenado en el disco duro.

    En la imagen se puede observar el funcionamiento de los enlaces duros, donde dentro del disco duro se encuentran los datos y los dos enlaces duros apuntan hacia una sola fuente.

  • Enlaces simbólicos: Los enlaces simbólicos apuntan al nombre de un archivo y posteriormente el archivo apunta a un contenido almacenado dentro del disco duro.
  • Directorios: Son archivos especiales que contienen referencias a otros archivos o directorios.
  • Archivos especiales: Suelen representar dispositivos físicos, como unidades de almacenamiento, impresoras, terminales, entre otros. En Linux, todo dispositivo físico que se conecte al ordenador está asociado a un archivo, el cual es tratado como un archivo ordinario.

Manipulación de archivos

Linux dispone de una serie de comandos para el manejo de archivos por medio de consola. Esto nos permite terner un mayor control, puesto que podemos realizar operaciones básicas sobre ellos como crear, copiar, mover, entre otros.

Comando Descripción Sintaxis Argumentos
touch Crea un archivo vacío, si este exite actualiza la hora de modificación touch <nombre_archivo>
cp Permite copiar un archivo a otro directorio cp <archivo_origen> <directorio_destino> -d: copia enlaces simbolicos.

-f: Fuerza a sobreescribir los archivos de destino si existe.

-i: Muestra un mensaje antes de sobreescribir los cambios.

-r: Copia directorios y sus archivos de forma recursiva.

-t: Se utiliza cuando se deseean copiar multiples archivos a un solo destino.
rm Elimina un archivo rm <ruta_archivo> -f: Forzar a la eliminación del archivo.

-i: Pregunta antes de eliminar el archivo.

-r: Elimina directorios y sus contenido.

-d: Elimina directorios vacíos.

-v: Muestra el proceso que se esta realizando.
mv Permite mover un archivo a otro directorio. Tip: Este comando sirve para renombrar archivos; simplemente moverlo a la misma carpeta con otro nombre mv <archivo_origen> <archivo_destino> -f: No pide permiso para mover el archivo.

-i: Pregunta si quiero mover el archivo.

-n: No sobreescribe si hay un archivo existente.
mkdir Permite la creación de directorios, si estos no existen mkdir <directorios> -p: permite la creación de directorios padres, si estos no existen.

-m: permite agregar permisos en la creacion del directorio.

-v: muestra detalles completos de la operación que está realizando.
cd Permite la navegacion entre directorios cd <ruta_a_navegar> / : ruta al directorio raiz.

~ : ruta al directorio de inicio.

.. : navegacion un directorio atras.
