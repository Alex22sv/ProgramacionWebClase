Manejo de archivos.

GNU/Linux.

Introducción.

La presente guía pretende introducir los conceptos genererales acerca del uso de archivos, especificamente en una distribución del sistema operativo GNU/Linux.

Un archivo es, en escencia, un conjunto de información, agrupada en determinada estructura. Sin importar el dispositivo que se utilize, toda la información en una computadora es almacenada como archivos. De acuerdo a su estructura, los archivos se clasifican en diversos tipos: audio, video, imagen, texto y muchos otros.

Características de los archivos

Todo archivo posee las siguientes características:

Nombre:
Su principal propósito es identificarlo de otros archivos y ayudar al usuario a conocer su contenido.
Extensión:
Es parte del nombre del archivo. Al final del nombre se coloca un punto seguido de unas pocas letras o números que definen la extensión. El uso de una extensiones es opcional en sistemas operativos basados en Unix.
Tamaño:
Es el espacio que el archivo necesita para ser almacenado en el dispositivo.
Tipo:
Indica el programa utilizado para acceder al contenido del archivo.
Fecha de modificación:
Es la última fecha en que el archivo fue modificado o creado.

Extensiones de archivos.

Las extensiones son únicamente un grupo de caracteres que, precedidos de un punto, son agregados al final del nombre del archivo.

En sistemas operativos Linux se agrega el tipo de archivo MIME al inicio de este, de manera que la extensión no es necesaria, sin embargo, la extensión es utilizada como ayuda al usuario para identificar el tipo de archivo. Si este no está claro la extensión es utilizada para determinarlo.Esto púede ocurrir cuando se transfieren archivos desde un sistema operativo Windows a Linux.

Los sistemas operativos Windows se basan únicamente en la extensión para saber el tipo de archivo; para evitar que usuarios modifiquen esta extensión, Windows oculta las extensiones de archivos de manera predeterminada. Esta opción puede ser cambiada en la configuración de usuario. Si se utiliza un sistema Windows se debe tener especial cuidado al crear o renombrar un archivo. Es común que no se pueda acceder a alguno debido a que este posee una extensión incorrecta o no posea ninguna.

Rutas.

Cuando se trabaja con comandos, es habitual pasar como parámetros archivos o directorios. Para indicar alguno de estos se utiliza la ruta o path del mismo, la cual que puede ser absoluta o relativa.

A diferencia de Windows, Linux no tiene unidades, por lo que no es necesario utilizar identificadores como A:\, C:\, D:\ ,etc. El sistema de archivos en Linux funciona con nodos, y cada nodo representa un archivo. El nodo principal puede ser accedido a través del caracter "/"; y el resto de particiones, discos duros, unidades de cd, unidades de dvd, o cualquier otro sistema de archivos, se deben acceder desde directorio principal como si fuese un directorio normal.

Las rutas absolutas se caracterizan por empezar siempre desde la raíz (es decir la "/"), y contener todos los directorios que hay desde este punto hasta el archivo o directorio que queremos indicar; sólo es posible escribir de una forma una ruta absoluta.

Todas las siguientes rutas son absolutas:

  • /home/DouglasHdezT/Desktop
  • /home/DouglasHdezT/Documents/Web/Archivos/index.html
  • /home/DouglasHdezT/Images/imagen_generica.jpg

En cambio las rutas relativas nunca empiezan por el caracter / y puede haber varias rutas relativas para el mismo archivo o directorio al que se quiere acceder. Esto es así porque la ruta relativa depende del directorio en el que se está trabajando.

Por ejemplo,si el directorio Imagenes contiene el archivo imagen_generica.jpg, se puede acceder a este archivo con diferentes rutas relativas de acuerdo con las siguientes situaciones:

Desde el directorio "Imagenes"
imagen_generica.jpg
Desde el directorio "DouglasHdezT"
Imagenes/imagen_generica.jpg
Desde el directorio "/"
home/DouglasHdezT/imagen_generica.jpg

Tipos de archivos.

En Linux existen, básicamente, 5 tipos de archivos:

  • Archivos ordinarios: Contienen la información con la que trabaja cada usuario.
  • Enlaces físicos o duros (hard links): Para lograr entender que es un enlace fisico, primero se debe comprender qué es un inodo. Un inodo es una estructura de datos, propia de los sistemas de archivos tradicionalmente empleados en los sistemas operativos tipo UNIX, como es el caso de Linux. Un inodo contiene las características de un archivo regular, directorio, o cualquier otro objeto que pueda contener el sistema de ficheros.

    • Una vez comprendido esto, un enlace duro es un archivo que apunta al mismo contenido almacenado en disco al que apunta el archivo original. Por lo tanto los archivos originales y los enlaces duros dispondrán del mismo inodo, y consecuentemente ambos estarán apuntando hacia el mismo contenido almacenado en el disco duro.

    En la imagen se puede observar el funcionamiento de los enlaces duros, donde dentro del disco duro se encuentran los datos y los dos enlaces duros apuntan hacia una sola fuente.

  • Enlaces simbólicos: Los enlaces simbólicos apuntan al nombre de un archivo y posteriormente el archivo apunta a un contenido almacenado dentro del disco duro.
  • Directorios: Son archivos especiales que contienen referencias a otros archivos o directorios.
  • Archivos especiales: Suelen representar dispositivos físicos, como unidades de almacenamiento, impresoras, terminales, entre otros. En Linux, todo dispositivo físico que se conecte al ordenador está asociado a un archivo, el cual es tratado como un archivo ordinario.

Manipulación de archivos.

Linux dispone de una serie de comandos para el manejo de archivos por medio de consola. Esto nos permite terner un mayor control, puesto que podemos realizar operaciones básicas sobre ellos como crear, copiar, mover, entre otros.

Comando Descripción Sintaxis Argumentos
touch Crea un archivo vacío, si este exite actualiza la hora de modificación touch <nombre_archivo>
cp Permite copiar un archivo a otro directorio cp <archivo_origen> <directorio_destino> -d: copia enlaces simbolicos.

-f: Fuerza a sobreescribir los archivos de destino si existe.

-i: Muestra un mensaje antes de sobreescribir los cambios.

-r: Copia directorios y sus archivos de forma recursiva.

-t: Se utiliza cuando se deseean copiar multiples archivos a un solo destino.
rm Elimina un archivo rm <ruta_archivo> -f: Forzar a la eliminación del archivo.

-i: Pregunta antes de eliminar el archivo.

-r: Elimina directorios y sus contenido.

-d: Elimina directorios vacíos.

-v: Muestra el proceso que se esta realizando.
mv Permite mover un archivo a otro directorio. Tip: Este comando sirve para renombrar archivos; simplemente moverlo a la misma carpeta con otro nombre mv <archivo_origen> <archivo_destino> -f: No pide permiso para mover el archivo.

-i: Pregunta si quiero mover el archivo.

-n: No sobreescribe si hay un archivo existente.
mkdir Permite la creación de directorios, si estos no existen mkdir <directorios> -p: permite la creación de directorios padres, si estos no existen.

-m: permite agregar permisos en la creación del directorio.

-v: muestra detalles completos de la operación que está realizando.
cd Permite la navegacion entre directorios cd <ruta_a_navegar> / : ruta al directorio raiz.

~ : ruta al directorio de inicio.

.. : navegacion un directorio atras.

Usuarios y grupos.

El objetivo principal de el uso de usuarios y grupos en un sistema Linux, es permitir que múltiples usuarios hagan uso del sistema, pero de una manera ordenada y segura. Asegurando que ninguna de las tareas realizadas por cualquier usuario pueda poner en riesgo todo el sistema. A través de la gestión de usuarios y grupos, se establecen los mecanismos y políticas de seguridad para la protección de los datos de cada usuario, así como para asegurar y proteger el funcionamiento de todo el sistema.

Usuarios.

Para poder usar el sistema operativo Linux, se es necesario tener una cuenta de usuario. Esta cuenta se compone por un nombre de usuario y de una contraseña. Las cuentas de usuarios son manejadas y creadas solo por el administrador del sistema, que se le conoce como el usuario root.

Cada usuario debe de pertenecer obligatoriamente a un grupo de usuario. Al momento de ingresar al sistema, el usuario debe de identificarse con su cuenta, y en caso de equivocarse el sistema le denegara el acceso. Luego de identificarse, el usuario podrá hacer uso del sistema y ejecutar todas las aplicaciones que tenga permitido, así como realizar acciones(leer, modificar o borrar) sobre los archivos en los que tenga permisos.

Por otra parte, una cuenta de usuario no solo provee un simple nombre, también es el punto de partida para poder establecer la ruta en donde se almacenan los documentos y el perfil del usuario. En Linux, generalmente es dentro de la carpeta de /home/nombre-usuario

Creación de usuarios.

Para poder añadir un usuario, se utiliza la consola (terminal), con el comando useradd. Su sintaxis es: 

                
    useradd [opciones] <nombre_usuario>

Donde, [opciones] puede ser remplazado por las siguientes:

  • -g: Grupo principal que sera asignado al usuario.
  • -d: Para asignar la carpeta home del usuario. Generalmente es /home/nombre-usuario
  • -m: Crear la carpeta home en caso de que esta no exista.
  • -s: Intérprete de comandos (shell) del usuario. Este suele ser /bin/bash

Por ejemplo, se quiere crear un usuario llamado "pedro" y que: su grupo inicial sea "instructores", tenga asignada como carpeta home "/home/pedro" y sus comandos sean interpretados en "/bin/bash"(lugar por defecto). El comando correcto que se debe ejecutar es el siguiente: 

                
    sudo useradd -g instructores -d /home/pedro -m Pedro

Ahora solo queda asignarle su contraseña, mediante el uso del comando passwd: 

                
    sudo passwd pedro

El sistema solicitará la contraseña dos veces y esta será asignada.

Dos cosas a tomar en cuenta:

  1. Todos estos comandos se ejecutan junto el comando sudo, ya que este permite al usuario actual, poder ejecutar programas con privilegios de seguridad del usuario root (solo si se le ha otorgado tales privilegios), y como se mencionó anteriormente, el único capaz de gestionar los usuarios es el administrador del sistema (usuario root)
  2. Varificar al momento de crear los nombres de usuario en minúsculas, porque para el sistema operativo Linux, "Pedro" es distinto que "pedro", ya que distingue entre mayúsculas y minúsculas. También se puede incluir el uso de números y algún signo como guión o guión bajo.

Modificación de usuarios.

Para poder realizar modificaciones sobre los usuarios, se utiliza el comando usermod. Este permite hacer cambios al nombre, a la carpeta home asignada, a su interpretador de comandos asignado, sus grupos, entre otros.

Por ejemplos, si se quisiera cambiar el nombre del usuario "pedro" a "obdulio" se realiza de la siguiente manera: 

                
    sudo usermod -l obdulio pedro

Eliminación de usuarios.

La eliminación de un usuario se realiza por medio de el uso del comando userdel seguido del nombre de usuario. Si se le añade la opción -r, su carpeta home respectiva también sera eliminada.

Por ejemplo, si se quisiera eliminar el usuario obdulio, se realiza de la siguiente manera: 

                
    sudo userdel obdulio

Grupos.

Para que exista una administración flexible de los permisos de los usuarios, el sistema operativo Linux permite catalogar a los usuarios a través de grupos y los permisos pueden ser asignados a un grupo. Por ejemplo, se cuenta con un grupo de instructores para una materia en específico, este grupo tiene acceso a ciertos archivos; al momento de incorporar a un nuevo instructor, solo se le asigna a la cuenta del instructor el grupo "instructores".

Como se mencionó anteriormente, todos los usuarios deben pertenecer a un grupo principal o primario obligatoriamente, pero puede pertenecer a otros grupos, a los cuales se les considera secundarios. Todos los grupos pueden contener varios usuarios, pero no pueden contener a otros grupos.

Cada grupo de usuario en el sistema Linux es identificado con un número diferente. A esto se le conoce como identificador de grupo o gid (Group IDentifier). El sistema, internamente realiza los trabajos bajo el gid y no bajo el nombre del grupo. Por lo general, cuando se crean los grupos, el sistema asigna un gid de "1000" en adelante, ya que los gid menores a "1000" están reservados para uso del sistema y sus grupos especiales.

Por defecto, la información de los grupos se guarda en el archivo /etc/group, el cual puede ser visualizado por cualquier editor de texto.

Creación de grupos.

Para poder realizar la creación, se hace mediante el comando groupadd, tan solo se debe indicar el nombre del grupo como parámetro. Su sintaxis es: 

                
    sudo groupadd <nombre_grupo>

Por ejemplo, si se quisiera crear un grupo llamado "alumnos", se realiza de la siguiente manera: 

                
    sudo groupadd alumnos

Modificación de grupos.

Los grupos también pueden ser modificados al igual que los usuarios. Para ellos, se hace uso del comando groupmod. Su sintaxis es: 

                
    sudo groupmod [opciones] <nombre-grupo>

En donde [opciones] puede ser remplazado por las siguientes:

  • -g: Sirve para poder cambiar el identificador de grupo.
  • -n: Sirve para renombrar el grupo.

Por ejemplo, si se quiere cambiar el nombre del grupo "alumnos" a "estudiantes", se realiza de la siguiente manera: 

                
    sudo groupmod -n estudiantes alumnos

Eliminación de grupos.

Para poder eliminar un grupo se hace por medio del comando groupdel seguido del nombre del grupo. El grupo se eliminará sólo en el caso de que no tenga usuarios con el grupo asignado como principal, si existe algun usuario con esta condición, el grupo no se eliminará.

Por ejemplo, si se quiere eliminar el grupo estudiantes, se realiza de la siguiente manera: 

                
    sudo groupdel estudiantes

Añadir usuarios a un grupo.

Para poder añadir usuarios a un grupo se hace por medio del comando adduser seguido del nombre del usuario y el nombre del grupo.

Por ejemplo, si se quiere agregar el usuario "roberto" al grupo de "estudiantes", se realiza de la siguiente manera: 

                
    sudo adduser roberto estudiantes

Quitar usuarios de un grupo.

Si se quiere eliminar un usuario de un grupo, utilizamos el comando deluser seguido del nombre del usuario y el nombre del grupo.

Por ejemplo, si queremos quitar al usuario "obdulio" del grupo de "instructores", se realiza de la siguiente manera: 

                
    sudo deluser obdulio instructores
