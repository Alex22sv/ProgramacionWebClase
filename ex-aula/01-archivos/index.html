Uso de archivos en Ubuntu

La presente es un guía ex-aula acerca del uso de archivos en el sistema operativo linux. El objetivo de esta es resolver de antemano algunas de las dudas que puedan surgir respecto al tema durante el desarrollo del curso.

Introducción

Un archivo es en escencia un conjunto de información, agrupada en determinada estructura. Sin importar el dispositivo que se utilize, toda la información en una computadora es almacenada como archivos. De acuerdo a su estructura los archivos se clasifican en tipos: audio, video, imagen, texto y muchos otros.

Características de archivos

Todo archivo posee las siguientes características:

Nombre
Su principal propósito es identificarlo de entre otros archivos y ayudar al usuario a conocer su contenido.
Extensión (opcional)
Es parte del nombre del archivo. Al final del nombre se coloca un punto seguido de unas pocas letras o números que definen la extensión.
Tamaño
Es el espacio que el archivo necesita para ser almacenado en el dispositivo.
Tipo
Indica el programa utilizado para acceder al contenido del archivo.
Fecha de modificación
Es la última fecha en que el archivo fué modificado o creado.

Extensiones de archivos

Las extensiones son únicamente un grupo de caracteres que, precedidos de un punto, son agregados al final del nombre del archivo.

En sistemas operativos linux se agrega el tipo de archivo MIME al inicio de este, de manera que la extensión no es necesaria sin embargo, la extensión es utilizada como ayuda al usuario para identificar el tipo de archivo. Si el tipo de archivo no está claro la extensión es utilizada para determinar el tipo de archivo, lo que puede ocurrir si se transfieren archivos desde un sistema operativo Windows a otro Linux.

Sistemas operativos Windows se basan únicamente en la extensión para saber el tipo de archivo y para evitar que usuarios modifiquen esta extensión oculta las extensiones de archivos de manera predeterminada, sin embargo, es posible cambiar esto. Si se utiliza un sistema Windows se debe tener especial cuidado al crear o renombrar un archivo. Es común que no se pueda acceder a un archivo debido a que este posee una extensión incorrecta o no posea ninguna.

Rutas

Cuando se trabaja con comandos es habitual tener que pasar como parámetros archivos o directorios. Para indicar un archivo o directorio se utiliza la ruta o path, que puede ser absoluta o relativa

A diferencia de Windows, Linux no tiene unidades, así que os podéis olvidar de A:\, C:\, D:\ y todas las letras del alfabeto. En Linux hay un sistema de archivos principal que es la raíz y se representa por el carácter /. El resto de particiones, pendrives, discos duros, unidades de cd, unidades de dvd y cualquier otra cosa que contenga un sistema de archivos, cuelga de algún directorio que esté en el sistema de archivos principal.

Las rutas absolutas se caracterizan por empezar siempre desde la raíz, es decir la /, y contener todos los directorios que hay desde la raíz hasta el archivo o directorio que queremos indicar, sólo es posible escribir de una forma una ruta absoluta. Todas las rutas siguientes son rutas absolutas por que empiezan desde la raíz. Algunos ejemplos serían:

  • /home/DHdezT/Desktop
  • /home/DHdezT/Documents/Web2/index.html
  • /home/DHdezT/Images/Foto-clan.jpg

Cuando tienen la / al final significa que la ruta es de un directorio, aunque no es necesaria la /. Cuando la ruta es de un archivo NUNCA se pone la / al final.

En cambio las rutas relativas nunca empiezan por la / y puede haber varias rutas relativas para el mismo archivo o directorio que queremos indicar. Esto es así por que la ruta relativa depende del directorio en el que se esté trabajando. Por ejemplo si estamos en el mismo directorio que el archivo Foto-clan.jpg, la ruta relativa es simplemente el nombre del archivo:

  • Foto-clan.jpg

En cambio, si nos encontramos dentro de la carpeta DHdezT del ejemplo de rutas absolutas; la ruta relativa para esta misma imagen sería:

  • Images/Foto-clan.jpg

Tipos de Archivos

En Linux existen básicamente 5 tipos de archivos:

  • Archivos ordinarios: Contienen la información con la que trabaja cada usuario.

  • Enlaces físicos o duros (hard links): Para lograr entender que es un enlace fisico, primero se debe comprender que es un inodo. Un inodo es una estructura de datos propia de los sistemas de archivos tradicionalmente empleados en los sistemas operativos tipo UNIX como es el caso de Linux. Un inodo contiene las características de un archivo regular, directorio, o cualquier otro objeto que pueda contener el sistema de ficheros.
    Una vez comprendido esto, un enlace duro es un archivo que apunta al mismo contenido almacenado en disco que el archivo original. Por lo tanto los archivos originales y los enlaces duros dispondrán del mismo inodo y consecuentemente ambos estarán apuntando hacia el mismo contenido almacenado en el disco duro.


  • Enlaces simbólicos: Los enlaces simbólicos apuntan al nombre de un archivo y posteriormente el archivo apunta a un contenido almacenado en el disco duro.

  • Directorios: Son archivos especiales que contienen referencias a otros archivos o directorios.

  • Archivos especiales: Suelen representar dispositivos físicos, como unidades de almacenamiento, impresoras, terminales, etc. En Linux, todo dispositivo físico que se conecte al ordenador está asociado a un archivo. Linux trata los archivos especiales como archivos ordinarios.

Permisos

Los permisos definen, de una manera simple y efectiva, reglas de acceso en lectura (r), escritura (w) o ejecución (x) a determinados archivos (si éstos no son ejecutables, el de ejecución no procede al igual que ocurre con las carpetas o directorios).

En un sistema Unix todos los archivos pertenecen obligatoriamente a un usuario y a un grupo. Al momento que un usuario crea un nuevo archivo, el propietario es el usuario creador y el grupo del archivo será el grupo principal del mismo usuario.

En la consola al utilizar el comandols –l(formato largo), Se puede visualizar el usuario propietario y el grupo propietario del archivo.

Tipo de permisos.

En los sistemas Unix, la gestión de los permisos que los usuarios y los grupos de usuarios tienen sobre los archivos y las carpetas, se realiza mediante un sencillos esquema de tres tipos de permisos:

  • Permisos de lectura
  • Permisos de escritura
  • Permisos de ejecución

Permisos de lectura.

Cuando un usuario tiene permiso de lectura de un archivo significa que puede leerlo o visualizarlo, bien sea con una aplicación o mediante comandos. Ejemplo, si tenemos permiso de lectura sobre el archivolabo.txt, significa que podemos ver el contenido del archivo. Si el usuario no tiene permiso de lectura, no podrá ver el contenido del archivo.

El permiso de lectura se simboliza con la letra'r' del inglés'read'.

Permiso de escritura.

Cuando un usuario tiene permiso de escritura sobre un archivo significa que puede modificar su contenido, e incluso borrarlo. También le da derecho a cambiar los permisos del archivo mediante el comandochmod así como cambiar su propietario y el grupo propietario mediante el comandochown. Si el usuario no tiene permiso de escritura, no podrá modificar el contenido del archivo.

El permiso de escritura se simboliza con la letra'w' del inglés'write'.

Permiso de ejecución.

Los permisos de ejecución se utilizan principalmente en aplicaciones y scripts. Si dispones de permisos puedes ejecutar la aplicación/script.

El permiso de ejecución se simboliza con la letra 'x' del inglés 'eXecute'.

Cambio de permisos de archivos en GNU LINUX.

En Linux, los archivos y ficheros utilizan permisos, los cuales pueden ser configurados o cambiados por el creador del del mismo, para impedir a otros usuarios de la red o del equipo, ver, ejecutar o escribir en este.

Como vimos anteriormente, los permisos que podemos cambiar o configurar en Linux son los siguientes: Lectura (r) el cual nos permitirá leer algún archivo o fichero, Escritura (w) que modifica un archivo o fichero y Ejecución (x) que ejecuta cualquier programa creado por un usuario.

Para el cambio de permisos nos apoyaremos con el uso del comando chmod, el cual posee una sintaxis de la siguiente forma: chmod ugo nombre_del_fichero

La parte de “ugo” del commando representa un numero en formato decimal que hace referencia a los permisos en sí. La letra U representa los permisos del usuario administrador, la letra G los del grupo al que pertenece, y la O a cualquier otro usuario no registrado en el grupo.

Serán 3 números que van del 0 al 7:

  • Primer número: permisos de propietario.
  • Segundo número: permisos de grupo.
  • Tercer número: permisos para el resto.

Para representar del 0 al 7 necesitamos 3 bits, que son los 3 bits utilizados para dar permisos a propietario, grupo y al resto. Con un '1' damos permiso y con un '0' se lo quitamos. Va en el siguiente orden: rwx.

A continuación, se especifican todas las combinaciones posibles de permisos (en casos particulares).

Binario Decimal Permiso Descripción
000 0 --- Ninguno
001 1 --x Ejecución
010 2 -w- Escritura
011 3 -wx Escritura y Ejecución
100 4 r-- Lectura
101 5 r-x Lectura y Ejecución
110 6 rw- Lectura y Escritura
111 7 rwx Lectura, Escritura y Ejecución

Algunos ejemplos de aplicación:

  1. 644:
    • Propietario: lectura y escritura.
    • Grupo: lectura.
    • Otros: lectura.
  2. 754:
    • Propietario: lectura, escritura y ejecución.
    • Grupo:lectura y ejecución
    • Otros: lectura.
  3. 777:
    • Propietario: lectura, escritura y ejecución.
    • Grupo: lectura, escritura y ejecución.
    • Otros: lectura, escritura y ejecución.

Usuarios y grupos

El objetivo principal de el uso de usuarios y grupos en un sistema Linux, es permitir que múltiples usuarios hagan uso del sistema, pero de una manera ordenada y segura.

De una manera que ninguna de las tareas realizadas por cualquier usuario pueda poner en riesgo todo el sistema. A través de la gestión de usuarios y grupos, se establecen los mecanismos y políticas de seguridad para la protección de los datos de cada usuario, así como para asegurar y proteger el funcionamiento de todo el sistema.

Usuarios.

Para poder usar el sistema operativo Linux, se es necesario tener una cuenta de usuario. Esta cuenta se compone por un nombre de usuario y de una contraseña.

Las cuentas de usuarios son manejadas y creadas solo por el administrador del sistema, que en el caso de ser un sistema operativo Linux a este se le conoce como el usuario root.

Cada usuario debe de pertenecer obligatoriamente a un grupo de usuario. Al momento de ingresar al sistema, el usuario debe de identificarse con su cuenta de usuario y en caso de equivocarse el sistema le denegara el acceso.

Luego de identificarse satisfactoriamente, el usuario podrá hacer uso del sistema y ejecutar todas las aplicaciones que tenga permitido, así como realizar acciones(leer, modificar o borrar) sobre los archivos en el que tenga permitido.

Por otra parte, una cuenta de usuario no solo provee un simple nombre, también es el punto de partida para poder establecer la ruta en donde se almacenan los documentos y el perfil del usuario. En linux, generalmente es dentro de la carpeta de /home/nombre-usuario

Creación de usuarios.

Para poder añadir un usuario, indicando sus parametros de información utilizamos en la consola (terminal) el comando useradd. Su sintaxis es:

  • useradd [opciones] nombre-usuario

Donde, [opciones] puede ser remplazado por las siguientes:

  • -g: Grupo principal que sera asignado al usuario.
  • -d: Para asignar la carpeta home del usuario. Generalmente es /home/nombre-usuario
  • -m: Crear la carpeta home en caso de que esta no exista.
  • -s: Intérprete de comandos (shell) del usuario. Este suele ser /bin/bash

Por ejemplo, se quiere crear un usuario llamado "pedro" y que su grupo inicial sea "instructores", que tenga asignada como carpeta home "/home/luis" y que sus comandos sean interpretados en "/bin/bash"(lugar por defecto). El comando correcto que se debe ejecutar es el siguiente:

  • sudo useradd -g instructores -d /home/pedro -m Pedro

Ahora solo nos queda asignarle su contraseña, mediante el uso del comando passwd:

  • sudo passwd pedro

El sistema nos solicitará la contraseña dos veces y listo, esta sera asignada.

Dos cosas a tomar en cuenta:

  1. Todos estos comandos se ejecutan junto el comando sudo, ya que este permite al usuario actual, poder ejecutar programas con privilegios de seguridad del usuario root (solo si se le ha otorgado tales privilegios), y como se mencionó anteriormente, el único capaz de gestionar los usuarios es el administrador (usuario root)
  2. Otra cosa a tomar en cuenta es el hecho de crear los nombres de usuario en minúsculas, porque para el sistema operativo Linux, "Pedro" es distinto que "pedro", ya que distingue entre mayúsculas y minúsculas. También se puede incluir el uso de números y algún signo como guión o guión bajo.

Modificación de usuarios.

Para poder realizar modificaciones sobre los usuarios, se utiliza el comando usermod. Este permite hacer cambios al nombre, a la carpeta home asignada, a su interpretador de comandos asignado, sus grupos, entre otros.

Por ejemplos, si se quisiera cambiar el nombre del usuario "pedro" a "obdulio" se realiza de la siguiente manera:

  • sudo usermod -l obdulio pedro

Eliminación de usuarios.

La eliminación de un usuario se realiza por medio de el uso del comando userdel seguido del nombre de usuario. Si se le añade la opción -r, su carpeta home respectiva también sera eliminada.

Por ejemplo, si se quisiera eliminar el usuario obdulio, se realiza de la siguiente manera:

  • sudo userdel obdulio

Grupos.

Para que exista una administración flexible de los permisos de los usuarios, el sistema operativo Linux permite catalogar los usuarios a través de grupos y los permisos pueden ser asignados a un grupo. Por ejemplo, se cuenta con un grupo de instructores para una materia en específico, este grupo tiene acceso a ciertos archivos, al momento de incorporar a un nuevo instructor, solo se le asigna a la cuenta del instructor el grupo "instructores".

Como se mencionó anteriormente, todos los usuarios deben pertenecer a un grupo principal o primario obligatoriamente, pero puede pertenecer a otros grupos, a los cuales se les considera secundarios. Todos los grupos de usuarios sólo pueden contener varios usuarios, pero no pueden contener a otros grupos.

Cada grupo de usuario en el sistema Linux es identificado con un numero diferente. A esto se le conoce como identificador de grupo o gid (Group IDentifier).. El sistema, internamente realiza los trabajos bajo el gid y no bajo el nombre del grupo. Por lo general, cuando se crean los grupos, el sistema asigna un gid de "1000" en adelante, ya que los gid menores a "1000" están reservados para uso del sistema y sus grupos especiales.

Por defecto en el sistema Linux, la información de los grupos se guarda en el archivo /etc/group, el cual puede ser visualizado por cualquier editor de texto.

Creación de grupos.

Para poder realizar la creación, se hace mediante el comando groupadd, tan solo se debe indicar el nombre del grupo como parámetro. Su sintaxis es:

  • sudo groupadd nombre-grupo

Por ejemplo, si se quisiera crear un grupo llamado "alumnos", se realiza de la siguiente manera:

  • sudo groupadd alumnos

Modificación de grupos.

Los grupos también pueden ser modificados al igual que los usuarios. Para ellos, se hace uso del comando groupmod. Su sintaxis es:

  • sudo groupmod [opción] nombre-grupo

En donde [opciones] puede ser remplazado por las siguientes:

  • -g: Sirve para poder cambiar el identificador de grupo.
  • -n: Sirve para renombrar el grupo.

Por ejemplo, si se quiere cambiar el nombre del grupo "alumnos" a "estudiantes", se realiza de la siguiente manera:

  • sudo groupmod -n estudiantes alumnos

Eliminación de grupos.

Para poder eliminar un grupo se hace por medio del comando groupdel seguido del nombre del grupo. El grupo se eliminara sólo en el caso de que no tenga usuarios con el grupo asignado como principal, si existe algun usuario con esta condición, el grupo no se eliminará.

Por ejemplo, si se quiere eliminar el grupo estudiantes, se realiza de la siguiente manera:

  • sudo groupdel estudiantes

Añadir usuarios a un grupo.

Para poder añadir usuarios a un grupo se hace por medio del comando adduser seguido del nombre del usuario y el nombre del grupo.

Por ejemplo, si se quiere agregar el usuario "roberto" al grupo de "estudiantes", se realiza de la siguiente manera:

  • sudo adduser roberto estudiantes

Quitar usuarios de un grupo.

Si se quiere eliminar un usuario de un grupo, utilizamos el comando deluser seguido del nombre del usuario y el nombre del grupo.

Por ejemplo, si queremos quitar al usuario "obdulio" del grupo de "instructores", se realiza de la siguiente manera:

  • sudo deluser obdulio instructores

Si se quiere conocer más acerca de todas las opciones disponibles de cada comando, se hace uso del comando man seguido del comando del que queremos.

Por ejemplo, si se quiere saber cuales son las opciones del comando usermod, se realiza de la siguiente manera:

  • man usermod

Asignación de grupos y usuarios a archivos.

Como se mencionó al principio, el objetivo principal de los usuarios y los grupos es obtener un uso óptimo del sistema, pero para eso se necesita poner en práctica el conocimiento acerca de los permisos en los usuarios y grupos.

Cambiar de propietario (chown)

Con el comando chown (CHange OWNer) se puede cambiar el usuario propietario de un fichero.

Para ejecutar este comando es necesario el uso de dos parámetros, el usuario que va a ser propietario y el objeto al que se le va a establecer la propiedad. Este comando es independiente de chmod, por lo que si a un fichero se le asigna un nuevo propietario, este nuevo porpietario conservará los permisos. Este comando solo lo puede ejecutar el usuario root.

Por ejemplo, se quiere que al fichero de nombre "notas" que se encuentra en la carpeta home del usuario nestor, se le asigne como nuevo propietario a obdulio, para realizarlo se hace de la siguiente manera:

  • sudo chown obdulio /home/nestor/notas

Cambiar de grupo propietario (chgrp)

Con el comando chgrp (CHange GRouP) se pude modificar al grupo propietario de un fichero.

Para poder ejecutar este comando es necesario el uso de dos parámetros, el grupo que va a ser propietario y el objeto al que se le va a establecer la propiedad. Este comando, al igual que el comando chown, es independiente de chmod, por lo que si a un fichero se le asigna un nuevo grupo propietario, este nuevo grupo propietario conservará los permisos.

Este comando está restringido, solo lo pueden usar los usuarios de forma limitada. Un usuario puede cambiar el grupo de un fichero si forma parte del nuevo grupo propietario.

Por ejemplo, se quiere que al fichero de nombre "laboratorios" que se encuentra en el directorio de documents que está en la carpeta home, se le asigne como nuevo grupo propietario a "instructores", para realizarlo se hace de la siguiente manera:

  • sudo chgrp instructores /home/documents/laboratorios
