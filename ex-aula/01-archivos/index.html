Uso de archivos en Ubuntu

La presente es un guía ex-aula acerca del uso de archivos en el sistema operativo linux. El objetivo de esta es resolver de antemano algunas de las dudas que puedan surgir respecto al tema durante el desarrollo del curso.

Introducción

Sin importar el dispositivo que se utilize, toda la información en una computadora es almacenada como archivos. Un archivo es en escencia un conjunto de información, agrupada en determinada estructura. De acuerdo a esta estructura los archivos se clasifican en tipos: audio, video, imagen, texto y muchos otros.

Características de archivos

Todo archivo posee las siguientes características:

Nombre
Su principal propósito es ayudar al usuario a identificar el contenido del archivo e identificarlo de entre otros archivos.
Extensión (opcional)
Es parte del nombre del archivo. Al final del nombre se coloca un punto seguido de unas pocas letras o números que definen la extensión del archivo.
Tamaño
Es el espacio que el archivo necesita para ser guardado en el almacenamiento del dispositivo.
Tipo
Indica el programa utilizado para acceder al contenido del archivo.
Fecha de modificación
Es la última fecha en que el archivo fué modificado o creado

Extensiones de archivos

La extensión es utilizada como ayuda al usuario para identificar el tipo de archivo. En sistemas operativos linux se agrega el tipo de archivo MIME al inicio de este, de manera que la extensión no es necesaria sin embargo, si el tipo de archivo no está claro la extensión es utilizada para determinar el tipo de archivo.

Si se está trabajando un sistema Windows se debe tener especial cuidado al crear o renombrar un archivo. Es común que no se pueda acceder a un archivo debido a que este posee una extensión incorrecta o no posea ninguna. Sistemas Windows se basan únicamente en la extensión para saber el tipo de archivo y para evitar que usuarios modifiquen esta extensión oculta las extensiones de archivos de manera predeterminada.

Tipos de Archivos

En Linux existen básicamente 5 tipos de archivos:

  • Archivos ordinarios: Contienen la información con la que trabaja cada usuario.

  • Enlaces físicos o duros (hard links): No es específicamente una clase de archivo sino un segundo nombre que se le da a un archivo. Supón que dos usuarios necesitan compartir información de un mismo archivo. Si cada uno tuviera una copia del archivo se soluciona el problema, pero las modificaciones que realice un usuario no las vería el otro. Sin embargo, si creamos un enlace duro al archivo para cada usuario cada vez que uno de ellos modifique cualquier cosa en el archivo, el otro lo podrá ver puesto que realmente están viendo y modificando el mismo archivo. El enlace sirve para localizar el archivo en su ubicación actual, pero no es el archivo real, sino un segundo nombre que se le da. De esta forma, con tener un solo archivo real este se podrá utilizar por todos los usuarios que lo necesiten mediante estos enlaces duros sin necesidad de duplicar o triplicar el archivo.

  • Enlaces simbólicos: También se utilizan para asignar un segundo nombre a un archivo. La diferencia con los enlaces duros es que los simbólicos solamente hacen referencia al nombre del archivo original, mientras que los duros hacen referencia al inodo en el que están situados los datos del archivo original. De esta manera, si tenemos un enlace simbólico y borramos el archivo original perderemos los datos, mientras que si tenemos un enlace duro los datos no se borrarán hasta que se hayan borrado todos y cada uno de los enlaces duros que existen hacia esos datos en el sistema de ficheros. El conteo del número de enlaces duros que tiene un fichero se realiza, como ya vimos, en el inodo correspondiente a los datos del fichero.

  • Directorios: Son archivos especiales que contienen referencias a otros archivos o directorios.

  • Archivos especiales: Suelen representar dispositivos físicos, como unidades de almacenamiento, impresoras, terminales, etc. En Linux, todo dispositivo físico que se conecte al ordenador está asociado a un archivo. Linux trata los archivos especiales como archivos ordinarios.

Manipulación de archivos

Linux dispone de una seria de comandos para el manejo de archivos por medio de consola. Esto nos permite terner una mayor control sobre ellos, puesto que podemos realizar operaciones mas complejas.

Comando Descripción Sintaxis Argumentos
cat Contatena varios archivos, para mostrarlos en pantalla. cat -argumentos ficheros -n: numerea el número líeas mostradas

-b: numerea el número líeas mostradas, pero no muestra las vacias.

-s: Elimina las lineas vacias seguidas, y las hace una sola.
touch Crea un archivo vacío, si este exite actualiza la hora de modificación touch archivo
cp Permite copiar un archivo a otro directorio cp archivo_origen directorio_destino -d: copia enlaces simbolicos.

-f: Fuerza a sobreescribir los archivos de destino si existe.

-i: Muestra un mensaje antes de sobreescribir los cambios.

-r: Copia directorios y sus archivos de forma recursiva.

-t: Se utiliza cuando se deseean copiar multiples archivos a un solo destino.
rm Elimina un archivo rm -argumento archivo -f: Forzar a la eliminación del archivo.

-i: Pregunta antes de eliminar el archivo.

-r: Elimina directorios y sus contenido.

-d: Elimina directorios vacíos.

-v: Mustra el proceso que se esta realizando.
mv Permite mover un archivo a otro directorio -f: No pide permiso para mover el archivo.

-i: Pregunta si quiero mover el archivo.

-n: No sobreescribe si hay un archivo existente.

Permisos

En un sistema Unix todos los archivos pertenecen obligatoriamente a un usuario y a un grupo. Al momento que un usuario crea un nuevo archivo, el propietario es el usuario creador y el grupo del archivo será el grupo principal del mismo usuario.

En la consola al utilizar el comando ls –l (formato largo), Se puede visualizar el usuario propietario y el grupo propietario del archivo.

Tipo de permisos.

En los sistemas Unix, la gestión de los permisos que los usuarios y los grupos de usuarios tienen sobre los archivos y las carpetas, se realiza mediante un sencillos esquema de tres tipos de permisos:

  • Permisos de lectura
  • Permisos de escritura
  • Permisos de ejecución

Permisos de lectura.

Cuando un usuario tiene permiso de lectura de un archivo significa que puede leerlo o visualizarlo, bien sea con una aplicación o mediante comandos. Ejemplo, si tenemos permiso de lectura sobre el archivo labo.txt, significa que podemos ver el contenido del archivo. Si el usuario no tiene permiso de lectura, no podrá ver el contenido del archivo.

El permiso de lectura se simboliza con la letra 'r' del inglés 'read'.

Permiso de escritura.

Cuando un usuario tiene permiso de escritura sobre un archivo significa que puede modificar su contenido, e incluso borrarlo. También le da derecho a cambiar los permisos del archivo mediante el comando chmod así como cambiar su propietario y el grupo propietario mediante el comando chown. Si el usuario no tiene permiso de escritura, no podrá modificar el contenido del archivo.

El permiso de escritura se simboliza con la letra 'w' del inglés 'write'.

Permiso de ejecución.

Cuando un usuario tiene permiso de ejecución de un archivo significa que puede ejecutarlo. Si el usuario no dispone de permiso de ejecución, no podrá ejecutarlo, aunque sea una aplicación.

Los únicos archivos ejecutables son las aplicaciones y los archivos de comandos (scripts). Si tratamos de ejecutar un archivo no ejecutable, dará errores.

El permiso de ejecución se simboliza con la letra 'x' del inglés 'eXecute'.
